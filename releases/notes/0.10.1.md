## 亮点
- Codex 审批与提问流程现在会在 app-server 刷新后保留待处理的 hook 介入状态，避免 UI 丢失 approve / deny / answer 操作入口。

- 补充 Codex 介入状态的回归测试，覆盖 hook 审批保留与刷新后的操作路径。

## Highlights
- Codex approval and question flows now preserve pending hook intervention state after app-server refreshes, keeping approve / deny / answer actions available in the UI.
- New installs show Claude / Codex usage headroom in the closed Island by default, making context and quota pressure visible immediately.

## Fixes
- Fixed Codex approval events that could be overwritten after refresh and stop responding to approval decisions.
- Added regression coverage for Codex intervention state preservation and refreshed approval paths.

## Notes
- The Sparkle build number is now `35`, and the app version is now `0.10.1`.
- This is a stability release based on `0.10.0`, focused on Codex approval state and default usage visibility.
