**日期：** 2026-05-18
**状态：** 待实现
**前置：** `docs/superpowers/specs/2026-05-18-island-plugin-protocol-design.md`（IPP v1）

## 一、背景

IPP v1 只开放了右耳给插件，且 `PluginSlot.compact` 不区分左右。v1.1 目标：

1. 细化 slot 声明（`compact-left` / `compact-right`）
2. 左耳在宠物 idle 时对插件开放
3. Settings 展示每个插件使用的 slot

## 二、区域模型（物理约束）

```
静默状态（无交互）:
┌──[左耳 ~22px]──[███物理刘海███]──[右耳 ~22px]──┐

hover / click 展开:
┌─────────────────────────────────────────────┐
│  [左耳]  [header: usage / 对话标题]  [右耳]  │
│                                              │
│         [展开面板内容]                        │
└─────────────────────────────────────────────┘
```

物理刘海遮住中间，静默状态只有左右两个耳朵可用。

## 三、Slot 枚举变更

### PluginSlot（`PingIsland/Services/Plugin/PluginModels.swift`）

```swift
enum PluginSlot: String, Codable, Equatable, Sendable {
    case compactLeft  = "compact-left"   // 左耳（宠物 idle 时可用）
    case compactRight = "compact-right"  // 右耳（常规插件 slot）
    case compact                          // 向后兼容，等同 compactRight
    case notification
    case expanded
}
```

**向后兼容规则：** manifest 里写 `"compact"` 的旧插件，`PluginHost` 在启动插件时将其等同于 `"compact-right"` 处理。`island/compact` 消息的 `position` 字段不变（仍用 `"left"` / `"right"`）。

### manifest.json 新写法示例

```json
{
  "id": "com.example.weather",
  "slots": ["compact-left", "notification", "expanded"]
}
```

旧写法 `"slots": ["compact"]` 仍有效。

## 四、左耳开放规则

| 状态 | 左耳显示 |
|------|---------|
| Claude 活动中（processing / waiting）| 宠物动画，插件不显示 |
| 宠物 idle（`showsClosedLeadingIcon == false`）| `pluginArbiter.activeLeft`（若有），否则空 |
| 岛已展开（opened）| 左耳随展开面板一起变化，不独立显示 compact 内容 |

`showsClosedLeadingIcon` 变化时同步调用 `PluginSlotArbiter.shared.setCoreActive(showing, side: .left)`，确保宠物有活动时插件内容自动让位。

## 五、NotchView 改动（`PingIsland/UI/Views/NotchView.swift`）

### 5.1 左耳插件渲染

在 `headerRow` 左侧 `MascotView` 的同级，添加插件 fallback：

```swift
// 当前（宠物显示时）
if viewModel.status != .opened && showsClosedLeadingIcon {
    MascotView(...)
        .frame(width: sideWidth)
}

// 新增：宠物不显示时，插件可占左耳
if viewModel.status != .opened && !showsClosedLeadingIcon,
   let pluginContent = pluginArbiter.activeLeft {
    IslandPluginRenderer.compactView(content: pluginContent)
        .frame(width: sideWidth)
        .onTapGesture {
            if let pluginId = pluginArbiter.activeLeftPluginId {
                viewModel.contentType = .plugin(pluginId: pluginId)
                viewModel.notchOpen(reason: .click)
            }
        }
}
```

### 5.2 左耳核心活动同步

在 `notchLayout` 或根视图加 `onChange`：

```swift
.onChange(of: showsClosedLeadingIcon) { _, showing in
    PluginSlotArbiter.shared.setCoreActive(showing, side: .left)
}
```

## 六、PluginsSettingsView 改动（`PingIsland/UI/Views/PluginsSettingsView.swift`）

插件行显示 slot badge，帮助用户理解插件占用哪些区域：

```
🌤  天气Demo   0.1.0   [左耳] [通知]         [开启]
⚡  系统监控   0.3.1   [右耳] [展开]         [开启]
```

Badge 从 `plugin.manifest.slots` 派生，使用小圆角标签样式：

```swift
ForEach(plugin.manifest.slots, id: \.self) { slot in
    Text(slot.displayName)
        .font(.system(size: 9, weight: .medium))
        .foregroundStyle(.secondary)
        .padding(.horizontal, 5)
        .padding(.vertical, 2)
        .background(.secondary.opacity(0.12), in: Capsule())
}
```

`PluginSlot.displayName` 扩展：

```swift
extension PluginSlot {
    var displayName: String {
        switch self {
        case .compactLeft:  return "左耳"
        case .compactRight, .compact: return "右耳"
        case .notification: return "通知"
        case .expanded:     return "展开"
        }
    }
}
```
